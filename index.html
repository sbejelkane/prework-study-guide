HTML

  • The head element contains information about the webpage.
    • The charset element sets our character encoding
    • The http-equiv element sets an HTTP header for content
    • The viewport element sets the default view for our webpage
    • The meta element defines metadata such as the character set, description, keywords, author, and viewport
      • metadata is data about our webpage that the browser uses to effectively serve our webpage to users; it contains important information about accessibility, search engines, and performance
  • The body element represents the visible content shown to the user.
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The script element embeds client-side scripts in the HTML document
  • HTML elements are CASE SENSITIVE. HTML declarations are not.
  • An HTML attribute defines additional information about an HTML element
  • The child elements are elements that are nested inside other elements
  • The parent element is the element that holds all the child elements
  • SEO: Search Engine Optimization
  • The footer element usually contains the author, copyright, contact, sitemap, and navigation

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch (called branch-name) and switches to it
  • git checkout main: switches to main branch
  • -b: flag that creates a new branch
  • add: command that adds modifications in the current branch to the staging area
  • -A: flag that indicates we want to add all changes
  • pull: command to receive a branch's modifications into the local environment
  • origin: keyword indicating the source of the pull
  • main: indicates the main/default branch
  • push: command to push local changes to GitHub branch
  • -m: flag used to associate a message with a commit

  • var keyword to declare a variable
  • console.log method used to output a message to the web console
  • if statement syntax if (condition) { //block of code to be executed if condition is truthy }
  • falsy / truthy A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy. 
  • if condition syntax if more than one if if (first condition) { // block of code to be executed if first condition is truthy } else if (second condition) { // block of code to be executed if second condition is truthy }
  • create an array just like declaring a variable, but use [ ] to surround the group, the data can be of different types within the same arrayvar shapes = ["triangle", "square", "pentagon", “circle”]; another example var studentInfo = [“Lu”, 54, true];
  • for loop A for loop contains the following three important statements
    • The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0), as shown in the following line of code:var x = 0;
    • The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property on our shapes array to express that condition, as shown in the following line of code:x < shapes.length;
    • The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1, as shown in the following line of code:x++
  • x++ same as x + 1
